Web Notebook Application

Designed a notebook web application in a Scrum team for the user to create, delete and update the accounts and the notes, including the functions to upload the attachments to the note, sent the email for the password reset process, and had the firewall to protect this application.

Used Spring Boot 2, Maven Framework, and RESTful API to create the application.
Used GitHub and followed the Forking Workflow.
Ran the application on the cloud with Tomcat 9.
Used AWS CloudFormation to create and terminate stacks.
Used AWS AMI to deploy AWS EC2.
Used AWS RDSwith MySQL database to store data.
Used CircleCI to implement the Continuous Deployment to make sure everything is up to date.
Used AWS CodeDeploy to Deploy the code and used AWS CloudWatch to monitor the status.
Used AWS Lambda and AWS DynamoDB to do the reset password process.
Used AWS AutoScaling to control the quantity of EC2.
Secured the Infrastructure with SSL Certificates.
Used AWS WAF as a firewall to protect our web application such as mitigate SQL Injection Attacks, mitigate cross-site scripting attach and Blacklist bad/hijacked JWT tokens or session IDs.
Implemented the Load test with JMeter.

Project Link: Web NoteBook(SpringBoot, AWS , CircleCI, JMeter)

Decentralized Weighted Voting System

Designed a decentralized voting application to achieve the trusted computing by identity management, ensure accountability, ensure non- repudiation and audit support, hence the voters can vote in the trustless environment with this system.

Used Solidity to create the smart contracts.
Used Truffle.js to deploy the smart contracts and used Web3.js to interact with the Blockchain.
Used Ganache as the local blockchain.
Used React Framework to create the front-end client site.
Used MetaMask to create the account wallet and make the transaction.
Used Express.js, Node.js and MongoDB Atlas to implement the login feature.

Project Link: Decentralized Weighted Voting System(Solidity, Web3.js, Truffle.js, MERN Stack)
